Cookie Consent Open Standard

Version 1.0 — A machine-readable declaration for cookie consent UIs

The Cookie Consent Open Standard defines a JSON file hosted at /.well-known/cookie-consent.json that describes a website's cookie consent implementation. Browser extensions, AI agents, and automated tools can read this file to understand and interact with cookie consent UIs without brittle CSS selector hacking.

Like robots.txt tells crawlers how to behave, cookie-consent.json tells consent tools how your cookie popup works.

Quick Start

The minimal valid file requires only two fields:

/.well-known/cookie-consent.json
{
  "version": "1.0",
  "categories": ["essential", "analytics", "marketing"]
}

That's it. Place this file on your web server and consent tools can already discover which cookie categories your site uses.

Full Schema

The complete specification with all optional fields:

Complete cookie-consent.json
{
  "version": "1.0",
  "categories": ["essential", "functional", "analytics", "marketing", "social-media"],
  "cmp": { "name": "onetrust", "version": "6.38.0" },
  "selectors": {
    "banner": "#onetrust-banner-sdk",
    "acceptAll": "#onetrust-accept-btn-handler",
    "rejectAll": "#onetrust-reject-all-handler",
    "preferences": "#onetrust-pc-btn-handler",
    "save": ".save-preference-btn-handler"
  },
  "api": {
    "type": "onetrust",
    "acceptAll": "OneTrust.AllowAll()",
    "rejectAll": "OneTrust.RejectAll()"
  },
  "gpc": true,
  "tcf": true,
  "contact": "privacy@example.com",
  "policyUrl": "https://example.com/cookie-policy"
}

Field Reference

FieldTypeRequiredDescription
versionstringYesSpec version, e.g. "1.0"
categoriesstring[]YesCookie categories used. Must include "essential".
cmpobjectNoCMP name and version
selectorsobjectNoCSS selectors for banner, buttons
categorySelectorsobjectNoPer-category toggle selectors and CMP IDs
apiobjectNoJavaScript API calls for consent actions
gpcbooleanNoWhether Global Privacy Control is respected
tcfbooleanNoWhether TCF v2 is implemented
contactstringNoPrivacy contact email
policyUrlstringNoURL to the cookie/privacy policy

Standard Categories

CategoryDescriptionExamples
essentialRequired for basic site functionalitySession ID, CSRF token, load balancer
functionalUser preferences and enhanced featuresLanguage, theme, chat widget state
analyticsAnonymous usage measurementGoogle Analytics, Plausible, Hotjar
marketingAd targeting and cross-site trackingFacebook Pixel, Google Ads, retargeting
social-mediaSocial sharing and embedded contentFacebook Like, Twitter embed, YouTube

Versioning

  • v1.0 -- initial spec (categories, optional selectors, optional CMP info)
  • v1.x -- backward-compatible additions (new optional fields only)
  • v2.0 -- reserved for breaking changes (new required fields)
  • Parsers must ignore unknown fields (forward-compatible)

Discovery Mechanisms

Tools can discover the file via any of these methods:

  1. Convention: fetch /.well-known/cookie-consent.json directly
  2. HTML meta: <link rel="cookie-consent" href="/.well-known/cookie-consent.json">
  3. robots.txt: add Cookie-Consent: /.well-known/cookie-consent.json
  4. llms.txt: reference in your AI instructions file
